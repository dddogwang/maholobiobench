MaholoBioBench · Research Introduction

面向生物实验室机器人的
数字孪生 Benchmark

定义实验室机器人动作任务的层次、种类、状态语义与判断标准，让生物实验操作可以被系统化评价。

Bio-Lab Manipulation Task Taxonomy Protocol State Evaluation Criteria
AutoMaholo / MaholoBioBench
这一页只需要讲清楚项目定位：不是做一个漂亮的仿真场景，而是把 Maholo 数字孪生变成可评价、可复现、可扩展的生物实验机器人 benchmark。

Motivation

实验室机器人缺少“生物实验语义”的动作任务基准

已有基础
已经证明什么
还缺什么
MaholoBioBench 的切入点
Maholo / LabDroid
iPSC 培养、细胞处理、DNA 提取等流程可以自动化执行
缺少面向学习和比较的标准任务定义
把真实实验动作拆成可执行 task families
Robot Benchmarks
reach、push、pick-place、door-open、语言长时序可被标准化评价
实验室设备、样品、液体和污染语义不足
加入 bio-lab protocol state
Lab Automation
实验规划、设备编排、实验日志可以形成自动化流程
机器人低层动作和实验协议之间缺少统一评分
区分几何成功和实验成功
这里强调 gap：普通 manipulation 成功不等于生物实验成功。比如机器人把管子放对位置，但拿错样品、重复使用 tip、体积错误，都应该算失败。

Evaluation

从 Geometric Success 到 Protocol Success

评分层级
判断问题
典型指标
为什么对生物实验重要
Robot / Geometry
动作有没有做成？
success rate、pose error、collision、joint limit
保证机器人基本可控和可执行
Protocol Progress
步骤有没有按协议推进？
completion rate、progress score、step order
长时序实验不能只看最终位置
Sample Correctness
是不是操作了正确样品？
sample id accuracy、container identity
拿错样品在几何上可能成功，但实验失败
Liquid Semantics
液体状态是否正确？
volume error、concentration error、source / target match

Bio-Lab Violations
有没有违反实验约束？
contamination、tip reuse、temperature zone、equipment state
污染和设备状态错误应直接扣分

一个策略可以“几何上成功”，但“实验上失败”。这正是 benchmark 要揭示的问题。

这里说明 evaluation 的新意。对于 benchmark paper，这些指标比单纯成功率更重要。
